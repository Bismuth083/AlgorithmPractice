# ビット演算

## 概要

- 内容: ビット演算の基本的な操作と応用方法について説明する。
- 前提知識: 基本的な演算、二進数、論理演算の基礎。

## 解説

### AND, OR, NOT, XOR 演算

積集合、和集合、補集合などの集合演算については、ここでは割愛する。不安な場合は、集合演算の解説を参照されたい。

整数型(`byte`, `int`, `ulong`)などに対してビット演算子を使うと、各桁ごとに論理演算が行われる。
(数値の前に`0b`というプレフィックスを付けると、二進数で数値を書くことができる)

```csharp

byte a = 0b_1100_1010;
byte b = 0b_1010_1100;

Console.WriteLine(~a); // -> 0b_0011_0101 (NOT演算)
Console.WriteLine(a & b); // -> 0b_1000_1000 (AND演算)
Console.WriteLine(a | b); // -> 0b_1110_1110 (OR演算)
Console.WriteLine(a ^ b); // -> 0b_0110_0110 (XOR演算)

```

念の為、各論理演算についても記述する。

- `~` (NOT演算): 1を0に、0を1に反転させる。
- `&` (AND演算): 両方のビットが1の場合にのみ1になる。
- `|` (OR演算): どちらかのビット、あるいは両方のビットが1の場合に1になる。
- `^` (XOR演算): 片方のビットが1で、もう片方のビットが0の場合に1になる。両方が同じ場合は0になる。

### シフト演算

シフト演算は、ビット列を左または右に移動させる操作である。つまり、桁をずらす操作である。

- `<<` (左シフト): ビット列を左に指定した数だけ移動させる。右側に0が追加される。
- `>>` (右シフト): ビット列を右に指定した数だけ移動させる。符号なし整数であれば、左側に0が追加される(論理シフトと言う)

```csharp

byte c = 0b_0011_1010;

Console.WriteLine(c << 2); // -> 0b_1110_1000 (左に2ビットシフト)
Console.WriteLine(c >> 2); // -> 0b_0000_1110 (右に2ビットシフト)

```

符号あり整数の場合、右シフト演算には2種類ある。

- `>>` (算術シフト): 最上位ビット(符号ビット)を保持したまま右にシフトする。つまり、負の数の場合は左側に1が追加される。
- `>>>` (論理シフト): C# 11以降で使用可能。符号あり整数に対しても、左側に0が追加される。

```csharp

sbyte d = 0b_1111_1000 ; // -8の意

Console.WriteLine(d >> 2); // -> 0b_1111_1110 (算術シフト、-2の意)
Console.WriteLine(d >>> 2); // -> 0b_0011_1110 (論理シフト、62の意)

```

普通の算術演算の場合では算術シフトの方が使われるが、ビット演算(あるいはフラグ)を行う場合には論理シフトの方が便利な場合が多い。
したがって、競技プログラミングでシフトを扱う場合には、符号なし整数型を使うことが多い(`byte`, `uint`, `ulong`など)。

### 列挙型によるフラグ管理

競技プログラミングのように、短期的に使うコードを書くときにはあまり必要がない可能性があるが、列挙型(`enum`)を使ってビットフラグを管理することもある。
これを使うとビット演算の可読性が向上するため、知っておいて損はない。

```csharp

enum Debuffs: byte{
  poisoned = 1 << 0, // 0b_0000_0001
  paralyzed = 1 << 1, // 0b_0000_0010
  fevered = 1 << 2, // 0b_0000_0100
  cursed = 1 << 3, // 0b_0000_1000
}

Debuffs currentDebuffs = Debuffs.poisoned | Debuffs.fevered; // 0b_0000_0101

```

### フラグ管理の基本操作

ビット演算を使うと、複数の状態を1つの整数で管理できる。これをフラグ管理と呼ぶ。  
(ここでいうi番目とは、1(2進数)の位を0番目として数えたもの)。



#### 確認系

|                     やりたいこと                     |           実装           |
| :--------------------------------------------------: | :----------------------: |
|       `num`に`i`番目のフラグがあるか確認する         |  `num & (1 << i) != 0`   |
|  `num`の`mask`部分が**全て**1になっているか確認する  |  `(bit & mask) == mask`  |
| `num`の`mask`の**いずれかが**1になっているか確認する |   `(bit & mask) != 0`    |

#### 設定系

|               やりたいこと                |       実装        |
| :---------------------------------------: | :---------------: |
|    `num`に`i`番目のフラグを**立てる**     | `num \| (1 << i)` |
|    `num`から`i`番目のフラグを**消す**     | `num & ~(1 << i)` |
|  `num`の`i`番目のフラグを**反転させる**   | `num ^ (1 << i)`  |
| `num`に`mask`部分のフラグを全て**立てる** |   `num \| mask`   |
| `num`から`mask`部分のフラグを**全て消す** |   `num & ~mask`   |
| `num`の`mask`部分のフラグを**反転させる** |   `num ^ mask`    |
|       `num`のフラグを**全て立てる**       |    `num = ~0`     |
|       `num`からフラグを**全て消す**       |     `num = 0`     |
|     `num`のフラグを**全て反転させる**     |      `~num`       |

#### その他

|                  やりたいこと                  |         実装          |                     例                     |
| :--------------------------------------------: | :-------------------: | :----------------------------------------: |
| 0番目**以上**i番目**未満**のフラグが立った数値 |    `(1 << i) - 1`     |       `(1 << 6) - 1 == 0b0011_1111`        |
| i番目**以上**j番目**未満**のフラグが立った数値 | `(1 << j) - (1 << i)` |    `(1 << 6) - (1 << 3) == 0b0011_1000`    |
|    `num`から`mask`部分のみを取り出した数値     |     `num & mask`      | `0b0101_1011 & 0b0000_1111 == 0b0000_1011` |
